# REVIEWPICK - AI 리뷰 큐레이션 쇼핑몰

## 서비스 개요
식품 쇼핑 시 수백 개의 리뷰를 직접 읽어야 하는 불편함을 해결하기 위해, AI가 리뷰를 관점별로 정리하여 한눈에 보여주는 서비스.
100개 식품 제품의 리뷰 10,000개를 수집하고, AI가 맛/가성비/건강 등 7개 관점으로 분류하여 사용자가 관심 있는 관점의 리뷰만 빠르게 탐색 가능.
자연어 검색을 통해 "가성비 좋은 라면", "아이 간식 추천" 같은 일상 언어로 원하는 제품과 리뷰를 바로 찾을 수 있음.

## 데이터 수집
쿠팡 식품 카테고리에서 인기 제품 100개를 선정하여 제품 정보(이름, 이미지, 가격)와 제품당 리뷰 최대 100개를 자동 수집.
봇 차단을 우회하기 위해 사람의 브라우징 패턴을 모사하는 방식(랜덤 대기, 자연스러운 스크롤)을 적용.
수집 중단 시 이어서 재개할 수 있도록 설계하여 대량 데이터 수집의 안정성을 확보.

## AI 전처리
수집된 리뷰 10,000개를 ChatGPT API에 전송하여, 각 리뷰에서 7개 관점(맛, 건강, 가성비, 다이어트, 조리 편의성, 가족/아이용, 재구매 의사)에 해당하는 핵심 문장을 추출.
동시에 100개 제품을 12개 음식 카테고리(라면, 과자, 음료, 신선식품 등)로 자동 분류.
이 전처리를 통해 사용자가 태그 하나만 클릭해도 제품별로 해당 관점의 리뷰 요약을 즉시 확인 가능.

## 검색 에이전트 구조
사용자의 자연어 검색어를 ChatGPT가 분석하여 두 가지 의도를 추출: "어떤 종류의 음식을 원하는가"(카테고리)와 "어떤 관점이 중요한가"(리뷰 태그).
추출된 카테고리로 제품을 필터링하고, 추출된 리뷰 태그로 관련 리뷰를 우선 노출하는 2단계 파이프라인.
예시: "건강한 간편식" 검색 → 냉동/간편식 카테고리 필터 + 건강 태그 리뷰 하이라이트.

## 한계 및 개선 방안
현재는 리뷰 관점 7개와 음식 카테고리 12개를 사전에 고정하고 단순 매칭하는 방식이라, 사용자 개인의 취향이나 맥락을 반영하지 못함.
개선안으로, 유저가 리뷰에 "좋아요"를 누르거나 특정 제품을 클릭한 행동 로그를 수집하고, 이를 기반으로 유저별 검색 모델을 학습시키는 개인화 검색 시스템 구현이 가능.
예를 들어 동일한 "간식 추천" 쿼리에도, 평소 건강 관점 리뷰에 관심이 많은 유저에게는 건강 리뷰를, 맛 위주로 탐색한 유저에게는 맛 리뷰를 우선 노출할 수 있음.
나아가 좋아요·클릭이 많이 축적된 리뷰와 제품을 전체 랭킹에 반영하면, 커뮤니티 기반의 리뷰 품질 필터링도 가능해짐.
